# ChatGPT Codex 客户端功能差距路线图

本路线图以项目内置 Codex `0.144.1` 的 CLI 功能和实验 app-server JSON Schema 为基准。目标是逐批接入可在桌面客户端稳定使用的能力，同时保留第三方 OpenAI 兼容接口。

## 已具备

- API Key、本地加密、多服务商与模型发现
- app-server 持久会话、恢复、分叉、归档、取消归档
- 会话重命名、永久删除、上下文压缩
- 轮次中断、执行中补充要求、交互审批和用户补充提问
- 图片附件、多代理状态、Token 统计
- 自动联网搜索与来源回答
- 插件市场、官方图标、安装卸载、自定义 MCP
- 定时任务、视频工作室、官方精选视频插件自动预装和离线 `$video-studio` Skill
- 当前工作区未提交改动的 Codex 代码审查
- API Key 与 ChatGPT 账户可选切换、套餐和额度展示
- 模型能力、推理强度、Personality、服务层级和运行时诊断
- GitHub Release 客户端更新、下载进度、更新前备份和历史恢复

## 兼容性原则

- 本地 app-server 能力优先走官方协议，不在前端模拟不存在的成功状态。
- 第三方接口不支持 Responses、工具调用或 Coding Plan 时自动回退，并明确说明缺失能力。
- ChatGPT 账号和 Remote Control 等能力需要对应账号或服务时，界面显示真实可用状态；API Key 始终作为默认入口。
- 每批修改都更新本路线图和根目录 `问题日志.md`。
