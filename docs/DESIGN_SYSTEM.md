# ChatGPT Codex 客户端设计系统

## 1. 视觉方向

延续当前 Codex 风格的深色侧边栏、暖灰主背景与荧光绿色强调色。视觉关键词：克制、专业、清晰、智能、可信。

## 2. 色彩

| Token | 建议值 | 用途 |
| --- | --- | --- |
| color-bg | #F4F5F2 | 应用主背景 |
| color-surface | #FFFFFF | 卡片与输入区域 |
| color-sidebar | #181A18 | 一级导航 |
| color-text | #22241F | 主文本 |
| color-muted | #858B82 | 次级文本 |
| color-border | #E1E5DD | 描边与分隔 |
| color-accent | #B9F46C | 主操作与运行状态 |
| color-purple | #8D7CF6 | Skills 与智能能力 |
| color-orange | #F2A65D | 媒体与提醒 |
| color-blue | #65ADF1 | 搜索与信息状态 |
| color-danger | #E36A62 | 错误与危险操作 |

强调色不用于大面积背景，主要用于当前状态、主按钮、进度和关键反馈。

## 3. 字体与层级

- 默认：Inter、PingFang SC、Microsoft YaHei。
- 页面标题：22–28px / 600–700。
- 卡片标题：14–16px / 600。
- 正文：12–14px / 400。
- 辅助说明：10–12px / 400。
- 数据指标：20–28px / 600–700。
- CLI：Cascadia Code 或 Consolas。

桌面端不要以过小文字换取信息密度。正式开发时正文下限建议为 12px。

## 4. 间距与尺寸

- 基础单位：4px。
- 页面外边距：24–32px。
- 卡片间距：12–16px。
- 卡片内边距：16–20px。
- 输入框高度：40–44px。
- 主按钮高度：40–44px。
- 侧边栏宽度：232–248px。
- 顶栏高度：60–64px。
- 圆角：按钮 8px，输入框 10px，卡片 12–16px，主弹窗 18–20px。

## 5. 组件规范

### 按钮
- Primary：用于当前页面唯一的主操作。
- Secondary：用于导出、筛选、管理等普通操作。
- Ghost：用于工具栏和轻量操作。
- Danger：删除任务、卸载插件、清除数据。
- Loading 状态禁止重复点击，并显示动作文本而不是只显示转圈。

### 输入框
- 默认展示简短示例，不使用术语堆叠。
- API Key 支持显示/隐藏、粘贴、格式检测和本地安全提示。
- 提示词输入支持附件、工具、模型和预估 Token。

### 卡片
- 同一页面内只使用一种主卡片样式。
- 可点击卡片必须有悬停、焦点和按下状态。
- 统计卡片优先展示结果，再展示趋势说明。

### 状态开关
- 开启状态使用绿色。
- 关闭状态使用中性灰。
- 不可用状态降低透明度，并必须解释原因。
- 开关不承担危险操作。

### 模态框
- 小型确认：400–480px。
- 表单任务：560–640px。
- 首次启动：880–960px。
- Esc 关闭；未保存表单关闭前提示。

## 6. 交互原则

1. 一个页面只有一个最强主操作。
2. 用户看到技术概念时必须同时看到通俗解释。
3. 自动选择必须可见、可更改、可撤销。
4. 请求进行中允许取消，并保留已完成步骤。
5. 所有异步操作必须提供状态反馈。
6. 错误信息说明“发生了什么、为什么、怎么解决”。
7. Token 与费用在请求前预估、请求中更新、请求后确认。

## 7. 动效

- 页面切换：120–180ms 淡入与轻微位移。
- 卡片悬停：最多上移 2px。
- 模态框：180–240ms 缩放与淡入。
- 流式输出：仅对新内容渐显，不持续闪烁。
- 任务执行：使用步骤进度，不使用无限旋转动画代替真实状态。
- 尊重系统“减少动态效果”设置。

## 8. 无障碍

- 所有按钮必须可键盘访问。
- 焦点态必须清晰可见。
- 正文与背景对比度不低于 WCAG AA。
- 不只使用颜色表达成功、失败或启停。
- 图标按钮必须有可读标签或 tooltip。
- 支持 125% 与 150% 系统缩放。

## 9. 桌面端适配

建议窗口最小尺寸为 1024 × 700；推荐尺寸为 1280 × 800 以上。

- 1024px 以下隐藏右侧任务上下文面板。
- 900px 以下侧边栏折叠为图标栏。
- 弹窗高度超过窗口时，表单主体内部滚动，标题和操作栏固定。
- 拖入文件时显示全窗口 Drop Zone。

## 10. 文案规范

优先使用用户语言：
- 使用“连接服务商”，避免“初始化 Provider”。
- 使用“每小时执行 3 次”，避免直接要求填写 Cron。
- 使用“安装能力”，避免只显示“拉取 MCP Server”。
- 使用“本次预计费用”，避免只显示原始 Token 数。

确认按钮描述动作结果，例如“创建任务”“连接并获取模型”“安装并启用”。


## Codex 原版配色基线

客户端使用 Codex 当前视觉语言中的黑白控件、低饱和灰色界面和柔和蓝紫强调，不再使用荧光绿色作为品牌主色。

| 用途 | 色值 |
| --- | --- |
| 主文字与主按钮 | `#111216` |
| 应用背景 | `#F6F7FB` |
| 卡片与输入框 | `#FFFFFF` |
| 左侧导航 | `#F0F1F5` |
| 分隔线 | `#DFE2EA` |
| 蓝紫强调 | `#5367E8` |
| 柔和蓝紫底色 | `#E8EDFF` |
| 成功状态绿 | `#6AA84F` |

使用规则：

- 黑色用于主操作、选中态和主要图标。
- 蓝紫色只用于联网、插件状态、链接和柔和背景提示。
- 绿色只表示连接成功、服务在线等语义状态。
- 欢迎页和空状态可使用浅蓝至蓝紫的柔和渐变。
- 大面积内容区保持白色或极浅灰色，确保代码与 Markdown 内容可读。
